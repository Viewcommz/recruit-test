# 요구사항 문서

## 소개

읽은 책 기억 도구는 사용자가 읽은 책에 대한 노트를 효율적으로 관리하고 검색할 수 있도록 하는 기능입니다. 기존의 책 관리 시스템에 독서 노트 입력, 검색, 필터링 기능을 추가하여 사용자가 읽은 내용을 쉽게 기억하고 찾을 수 있도록 합니다.

## 요구사항

### 요구사항 1

**사용자 스토리:** 독서가로서, 읽은 책에 대한 상세한 노트를 작성하고 저장할 수 있기를 원한다. 그래야 나중에 중요한 내용을 다시 찾아볼 수 있다.

#### 승인 기준

1. WHEN 사용자가 독서 노트 작성 페이지에 접근하면 THEN 시스템은 노트 입력 폼을 표시해야 한다
2. WHEN 사용자가 책 제목, 저자, 노트 내용, 태그를 입력하고 저장 버튼을 클릭하면 THEN 시스템은 노트를 데이터베이스에 저장해야 한다
3. WHEN 노트 저장이 성공하면 THEN 시스템은 성공 메시지를 표시하고 노트 목록 페이지로 이동해야 한다
4. IF 필수 필드(책 제목, 노트 내용)가 비어있으면 THEN 시스템은 유효성 검사 오류 메시지를 표시해야 한다

### 요구사항 2

**사용자 스토리:** 독서가로서, 저장된 독서 노트들을 키워드로 검색할 수 있기를 원한다. 그래야 특정 주제나 내용을 빠르게 찾을 수 있다.

#### 승인 기준

1. WHEN 사용자가 검색 입력 필드에 키워드를 입력하면 THEN 시스템은 실시간으로 검색 결과를 표시해야 한다
2. WHEN 검색어가 입력되면 THEN 시스템은 책 제목, 저자, 노트 내용에서 해당 키워드를 포함한 노트들을 반환해야 한다
3. WHEN 검색 결과가 없으면 THEN 시스템은 "검색 결과가 없습니다" 메시지를 표시해야 한다
4. WHEN 검색어를 지우면 THEN 시스템은 전체 노트 목록을 다시 표시해야 한다

### 요구사항 3

**사용자 스토리:** 독서가로서, 태그별로 독서 노트를 필터링할 수 있기를 원한다. 그래야 특정 주제나 카테고리의 노트들을 모아서 볼 수 있다.

#### 승인 기준

1. WHEN 사용자가 태그 필터 드롭다운을 클릭하면 THEN 시스템은 사용 가능한 모든 태그 목록을 표시해야 한다
2. WHEN 사용자가 특정 태그를 선택하면 THEN 시스템은 해당 태그가 포함된 노트들만 표시해야 한다
3. WHEN 여러 태그를 선택하면 THEN 시스템은 선택된 태그 중 하나라도 포함된 노트들을 표시해야 한다
4. WHEN "모든 태그" 옵션을 선택하면 THEN 시스템은 전체 노트 목록을 표시해야 한다

### 요구사항 4

**사용자 스토리:** 독서가로서, 저장된 독서 노트를 수정하고 삭제할 수 있기를 원한다. 그래야 노트 내용을 업데이트하거나 불필요한 노트를 제거할 수 있다.

#### 승인 기준

1. WHEN 사용자가 노트 목록에서 특정 노트의 수정 버튼을 클릭하면 THEN 시스템은 해당 노트의 편집 폼을 표시해야 한다
2. WHEN 사용자가 노트 내용을 수정하고 저장하면 THEN 시스템은 변경사항을 데이터베이스에 업데이트해야 한다
3. WHEN 사용자가 노트의 삭제 버튼을 클릭하면 THEN 시스템은 삭제 확인 대화상자를 표시해야 한다
4. WHEN 사용자가 삭제를 확인하면 THEN 시스템은 해당 노트를 데이터베이스에서 제거하고 목록을 업데이트해야 한다

### 요구사항 5

**사용자 스토리:** 독서가로서, 독서 노트 목록을 날짜순으로 정렬하여 볼 수 있기를 원한다. 그래야 최근에 작성한 노트나 오래된 노트를 쉽게 찾을 수 있다.

#### 승인 기준

1. WHEN 사용자가 노트 목록 페이지에 접근하면 THEN 시스템은 기본적으로 최신 작성일순으로 노트들을 표시해야 한다
2. WHEN 사용자가 정렬 옵션을 "오래된 순"으로 변경하면 THEN 시스템은 가장 오래된 노트부터 표시해야 한다
3. WHEN 사용자가 정렬 옵션을 "제목순"으로 변경하면 THEN 시스템은 책 제목의 알파벳/가나다 순으로 노트들을 표시해야 한다
4. WHEN 정렬 옵션이 변경되면 THEN 시스템은 현재 적용된 검색/필터 조건을 유지하면서 정렬만 변경해야 한다

### 요구사항 6

**사용자 스토리:** 개발자로서, 독서 노트 데이터를 안전하게 저장하고 조회할 수 있는 API를 제공받고 싶다. 그래야 프론트엔드에서 노트 기능을 구현할 수 있다.

#### 승인 기준

1. WHEN POST /api/notes 요청이 유효한 노트 데이터와 함께 전송되면 THEN 시스템은 노트를 생성하고 201 상태코드와 함께 생성된 노트 정보를 반환해야 한다
2. WHEN GET /api/notes 요청이 전송되면 THEN 시스템은 모든 노트 목록을 200 상태코드와 함께 반환해야 한다
3. WHEN GET /api/notes?search=키워드 요청이 전송되면 THEN 시스템은 키워드가 포함된 노트들만 필터링하여 반환해야 한다
4. WHEN GET /api/notes?tags=태그1,태그2 요청이 전송되면 THEN 시스템은 해당 태그들이 포함된 노트들만 필터링하여 반환해야 한다
5. WHEN PUT /api/notes/:id 요청이 유효한 업데이트 데이터와 함께 전송되면 THEN 시스템은 해당 노트를 업데이트하고 200 상태코드와 함께 업데이트된 노트 정보를 반환해야 한다
6. WHEN DELETE /api/notes/:id 요청이 전송되면 THEN 시스템은 해당 노트를 삭제하고 204 상태코드를 반환해야 한다
