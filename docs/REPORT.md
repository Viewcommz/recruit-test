## 1. 개발 결과 요약

### 선택한 문제와 해결 방법
**"읽고 싶은 책이 많은데 우선순위 결정을 못한다"** 문제를 선택했습니다. 현대인들은 읽고 싶은 책은 많지만 시간이 한정적이어서 어떤 책을 먼저 읽을지 결정하기 어려운 상황을 해결하고자 했습니다.

**핵심 해결 아이디어**: 페이지수와 출간일을 복합적으로 고려한 우선순위 알고리즘을 개발했습니다. 짧은 책일수록, 최신 출간일일수록 높은 우선순위를 부여하여 빠른 완독과 트렌드 반영을 동시에 고려했습니다.

### 구현한 주요 기능

1. **스마트 우선순위 계산 시스템** ⭐
   - 페이지수 기반 우선순위 (적을수록 높은 우선순위)
   - 출간일 기반 우선순위 (최신일수록 높은 우선순위)  
   - 복합 알고리즘: `(페이지수 * -1) + 출간일시간값`

2. **완전한 책 관리 시스템**
   - 책 추가/조회 기능 (제목, 저자, 페이지수, 출간일, 설명, 카테고리)
   - 실시간 데이터베이스 연동 (SQLite + Better-SQLite3)
   - RESTful API 설계 (Express.js + TypeScript)

3. **다양한 정렬 및 검색 기능**
   - 5가지 정렬 옵션: 우선순위, 제목순, 저자순, 페이지수, 출간일
   - 통합 검색 (제목, 저자, 설명 동시 검색)
   - 저자별, 카테고리별 필터링
   - 실시간 검색 결과 반영

4. **사용자 친화적 UI/UX**
   - 반응형 디자인 (모바일/데스크톱 대응)
   - 직관적 인터페이스 (이모지와 색상 활용)
   - 실시간 피드백 (로딩 상태, 성공/실패 메시지)
   - Next.js 14 + Tailwind CSS 기반

### 완성도/만족도

**완성도: 85%** 
- 핵심 기능은 모두 완성되어 실제 사용 가능한 수준
- 백엔드 API, 프론트엔드 UI, 데이터베이스 모두 완전 구현
- 우선순위 알고리즘이 의도한 대로 정확히 작동

**만족도: 90%**
- 단순한 CRUD를 넘어선 실용적인 우선순위 알고리즘 구현에 만족
- 복합 정렬 로직이 실제로 유용한 결과를 제공
- 확장 가능한 구조로 설계하여 향후 AI 추천 시스템 등 추가 기능 개발 용이
- 사용자 경험을 고려한 직관적인 인터페이스 완성

## 2. 개발 과정에서 느낀 점/회고

-   개발 중 어려웠던 점: 프로젝트 구조를 이해하고 AI를 어떻게 활용해야하는지 알아가는 시간이 생각보다 오래 소요되서 주어진 과제를 수행하는데 생각보다 시간이 부족했습니다.
- 어느정도 진행 방식을 파악하고 적극적으로 AI를 활용하는 과정에서 요청사항이 많아 오류를 자주 뱉어 시간 손실이 있었습니다.
- AI를 활용해서 프론트 페이지까지 빠르게 프로토타이핑 하는 과정을 IDE 내부에서 겪으니 확실히 편했습니다.
