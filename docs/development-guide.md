# 📚 1시간 AI 개발 테스트

## 🎯 테스트 개요

**소요시간**: 60분 (정확히 1시간)  
**테스트 형태**: 개인 풀스택 프로토타이핑 과제  
**평가 목적**: AI 도구 활용한 빠른 풀스택 개발 능력 측정

## 📋 문제

당신은 다음과 같은 상황에 놓여 있습니다:

```
📖 상황:
책을 좋아하는 사람들이 공통적으로 겪는 4가지 문제가 있습니다.

1. 읽고 싶은 책은 많은데 뭘 먼저 읽을지 결정하기 어렵다
2. 읽은 책의 내용이 기억나지 않는다
3. 친구들이 어떤 책을 읽고 있는지 알고 싶다
4. 자신의 독서 패턴을 분석해보고 싶다

🎯 과제:
위 4가지 문제 중 1-2개를 선택하여 해결하는
풀스택 웹 애플리케이션을 만드시오.
```

## ⚡ 제약사항

### 시간 제약

-   **정확히 60분** 안에 완성
-   시간 초과 시 미완성 상태로 제출

### 기술 제약

-   **프론트엔드**: Next.js 14 + TypeScript (필수)
-   **백엔드**: Express.js + TypeScript (필수)
-   **데이터 저장**: 메모리 저장 또는 간단한 JSON 파일만 허용
-   **외부 API**: 사용 금지
-   **라이브러리**: 꼭 필요한 경우에만 최소한 사용

### 범위 제약

-   **핵심 기능**: 1-2개만 선택하여 완성도 높게 구현
-   **API 설계**: 3-5개 엔드포인트
-   **UI**: 기본적인 사용성 확보 수준
-   **연동**: 프론트엔드-백엔드 정상 통신 필수

## 📤 제출 결과물

### 필수 제출물

1. **작동하는 풀스택 애플리케이션**

    - `npm run dev`로 실행 가능한 상태
    - 프론트엔드(3000)와 백엔드(8000) 모두 정상 작동
    - API 연동이 정상적으로 동작해야 함

2. **기획서** (`PLANNING.md`)

    - 선택한 문제와 해결 방법
    - 프론트엔드/백엔드 역할 분담
    - API 엔드포인트 설계 (3-5개)
    - 예상 사용 시나리오

3. **AI 활용 리포트** (`AI_USAGE.md`)
    - 사용한 AI 도구 목록
    - 프론트엔드/백엔드별 주요 질문(프롬프트) 3-5개
    - AI 활용으로 절약된 시간 추정

### 평가 요소

| 항목            | 비중 | 평가 기준                              |
| --------------- | ---- | -------------------------------------- |
| **창의적 기획** | 25%  | 문제 선택의 독창성, 해결 방법의 참신함 |
| **AI 활용도**   | 35%  | AI 도구 활용 효율성, 프롬프트 품질     |
| **풀스택 구현** | 30%  | 프론트엔드-백엔드 연동, API 설계 품질  |
| **코드 품질**   | 10%  | 가독성, 구조화 정도                    |

## 🚀 시작 방법

```bash
# 프로젝트 디렉토리 이동
cd recruit-test

# 모노레포 전체 실행 (프론트엔드 + 백엔드)
npm run dev
```

**확인사항**:

-   프론트엔드: http://localhost:3000 접속 가능
-   백엔드: http://localhost:8000/api/health 응답 확인
-   페이지에서 "백엔드 서버: 연결됨" 상태 확인

## ⏰ 시간 관리

**권장 시간 배분** (참고용):

-   기획: 5-10분
-   백엔드 API 구현: 15-20분
-   프론트엔드 구현: 15-20분
-   연동 및 테스트: 5-10분
-   정리: 5분

**시간 체크포인트**:

-   15분: API 엔드포인트 1-2개 완성
-   30분: 백엔드 기본 구조 완성
-   45분: 프론트엔드-백엔드 연동 완료
-   55분: 최종 테스트 및 정리
-   60분: 제출 준비 완료
