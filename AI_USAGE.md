## 주요 프롬프트 & 결과

### 1️⃣ Express API 설계

**사용 목적**: 기본적인 도서 리스트 API 스켈레톤을 빠르게 작성하기 위해
**프롬프트**:
``` 
Express.js로 책 우선순위 목록을 저장하는 POST /api/books 엔드포인트와 목록을 조회하는 GET /api/books 엔드포인트 코드를 TypeScript로 작성해줘.
```
**결과**: 생성된 코드를 수정하여 백엔드의 기본 구조를 완성했으며, 로컬 메모리 배열을 사용하도록 적용했다.
**만족도**: ⭐⭐⭐⭐☆

### 2️⃣ Next.js 컴포넌트 작성

**사용 목적**: 책 추가 폼과 목록 컴포넌트의 초기 틀을 만들기 위해
**프롬프트**:
```
Next.js 14 app router 기준으로 책 제목과 작가, 관심도, 긴급도를 입력받아 목록을 보여주는 컴포넌트를 작성해줘. Tailwind CSS 사용.
```
**결과**: 제공된 코드를 바탕으로 상태 관리와 스타일을 보완하여 원하는 UI를 빠르게 구성했다.
**만족도**: ⭐⭐⭐⭐⭐

### 3️⃣ 로직 검증 및 버그 수정

**사용 목적**: 우선순위 계산 로직에 오류가 없는지 확인하고 테스트 케이스 예시를 얻기 위해
**프롬프트**:
```
JavaScript에서 interest와 urgency를 받아 priority를 계산하는 함수를 작성하고 간단한 테스트 코드를 보여줘.
```
**결과**: AI가 제공한 예시를 기반으로 계산 함수를 구현하고 프론트엔드에 적용했다.
**만족도**: ⭐⭐⭐⭐☆

## AI 활용 효과

### ✅ 도움된 부분

1. **코드 생성 속도**: 기본 API와 컴포넌트 구조를 빠르게 생성해 주어 전체 개발 시간이 단축됨
2. **TypeScript 타입 정의**: 데이터 구조 정의 시 AI가 타입 예시를 제공해 오류를 줄임
3. **버그 해결**: 오류 메시지를 전달해 해결 방법을 제안받음

### ❌ 한계점 & 직접 구현한 부분

1. **세부 로직 설계**: 우선순위 계산 방식과 UI 흐름은 직접 고민하여 결정해야 했음
2. **사용성 개선**: 입력 검증 및 상세 UX는 AI가 제안하지 못해 직접 수정
3. **창의적 아이디어**: 기능 선정과 전체 플로우 구상은 스스로 진행
