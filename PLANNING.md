# 📋 기획서

## 선택한 문제

- [x] 📖 읽고 싶은 책 우선순위
- [x] 📚 읽은 책 기억 도구
- [ ] 👥 친구들 독서 현황
- [ ] 📊 독서 패턴 분석

**선택 이유**:
읽고 싶은 책을 우선순위로 정리하고 읽은 책의 내용을 기록하는 것이 가장 기본적인 독서 관리라고 판단했습니다. 제한된 시간 안에 구현하기 쉽고 사용 가치도 높습니다.

## 핵심 기능 (3-4개)

1. **책 추가 및 우선순위 저장**: 제목, 작가, 관심도 등을 입력하여 리스트에 저장
2. **우선순위 정렬**: 관심도와 긴급도를 이용해 자동 정렬된 목록 제공
3. **독서 노트 작성**: 읽은 책에 대한 짧은 메모와 태그를 남길 수 있음
4. **노트 검색**: 키워드나 태그로 노트를 빠르게 찾아보기 (선택사항)

## 해결 방법

**데이터 구조**:

```typescript
interface Book {
  id: string;
  title: string;
  author: string;
  priority: number; // 계산된 우선순위
}

interface ReadingNote {
  id: string;
  bookId: string;
  content: string;
  tags: string[];
  createdAt: string;
}
```

**핵심 알고리즘/로직**:

- 관심도(1-5)와 긴급도(1-5)를 입력받아 `priority = interest * 0.6 + urgency * 0.4` 방식으로 계산
- 우선순위가 높은 순으로 목록을 정렬
- 노트는 책 ID를 기준으로 연결하여 검색 가능하게 관리

**기술 스택**:

- Frontend: Next.js 14 + TypeScript + Tailwind CSS
- Backend: Express.js + TypeScript
- 데이터: localStorage (백엔드는 헬스체크만 활용)
- 추가 라이브러리: 필요 시 uuid

## 기대 효과

**사용자에게 제공할 가치**:

- 읽고 싶은 책을 체계적으로 정리하고 무엇을 먼저 읽을지 쉽게 결정할 수 있음
- 읽은 책의 주요 내용을 메모하여 나중에 빠르게 회고 가능

**차별화 포인트**:

- 복잡한 기능 없이 빠르게 입력하고 정렬할 수 있는 단순한 UI
- 검색 기능을 통해 메모를 빠르게 찾아볼 수 있음
