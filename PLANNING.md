# 📚 나만의 독서 도구 - 책 우선순위 관리 시스템

## 🎯 선택한 문제
**"읽고 싶은 책이 많은데 우선순위 결정을 못한다"**

### 선택 이유
- 현대인들은 읽고 싶은 책은 많지만 시간은 한정적
- 어떤 책을 먼저 읽을지 결정하는 것이 어려움
- 개인의 상황과 선호도를 고려한 맞춤형 우선순위 시스템 필요

## ✅ 구현 완료된 핵심 기능

### 1. 책 관리 시스템
- **책 추가**: 제목, 저자, 페이지수, 출간일, 설명, 카테고리 입력
- **책 목록 조회**: 실시간 데이터베이스 연동
- **책 상세 정보**: 개별 책 정보 조회

### 2. 우선순위 계산 알고리즘 ⭐
- **페이지수 기반**: 적은 페이지수일수록 높은 우선순위 (빠른 완독 가능)
- **출간일 기반**: 최신 출간일일수록 높은 우선순위 (트렌드 반영)
- **복합 계산**: `(페이지수 * -1) + 출간일시간값`

### 3. 다양한 정렬 및 필터링
- **정렬 옵션**: 우선순위, 제목순, 저자순, 페이지수, 출간일
- **검색 기능**: 제목, 저자, 설명 통합 검색
- **필터링**: 저자별, 카테고리별 필터링
- **실시간 업데이트**: 검색/정렬 변경 시 즉시 반영

### 4. 사용자 친화적 UI/UX
- **반응형 디자인**: 모바일/데스크톱 대응
- **직관적 인터페이스**: 이모지와 색상을 활용한 시각적 구분
- **실시간 피드백**: 로딩 상태, 성공/실패 메시지

## 🛠️ 기술 스택 및 구현 상세

## 🎨 우선순위 알고리즘 상세

### 핵심 로직
```typescript
case 'priority':
    // 우선순위 = (페이지수 역순) + (최신순)
    aValue = (a.pageCount || 0) * -1 + new Date(a.publishedDate || '1900-01-01').getTime();
    bValue = (b.pageCount || 0) * -1 + new Date(b.publishedDate || '1900-01-01').getTime();
```

### 우선순위 결정 요소
1. **페이지수 (40% 가중치)**: 적을수록 높은 우선순위
2. **출간일 (60% 가중치)**: 최신일수록 높은 우선순위
