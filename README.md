# 📚 AI 활용 풀스택 개발 테스트 - 나만의 독서 도구

## 테스트 개요

이 테스트는 Next.js(프론트엔드)와 Express.js(백엔드)를 활용하여, 실제 서비스처럼 문제를 정의하고, **Cursor**(AI 코딩 어시스턴트)만을 적극 활용해 빠르게 프로토타입을 완성하는 능력을 평가합니다.

-   **목표**: 창의적 문제 해결, AI 협업, 풀스택 구현 역량
-   **주요 도구**: Next.js, Express.js, TypeScript, Tailwind CSS, **Cursor**
-   **문제 예시**: 책 우선순위, 독서 기록, 친구 현황, 독서 패턴 분석 등 중 1~2개 선택

---

## 진행 절차

1. **저장소 환경설정**

    ```bash
    cd recruit-test
    npm install
    npm run dev
    ```

2. **PLANNING.md 작성**

    - 개발 시작 전, 본인이 선택한 문제와 해결 전략을 간단히 정리
    - 주요 작성 항목:
        - 어떤 문제를 선택했는지, 선택 이유
        - 구현할 핵심 기능(3~4개)
        - 데이터 구조/주요 API 설계
        - 기대 효과 및 차별화 포인트

3. **브랜치 전략**

    - dev 브랜치에서 직접 작업/커밋/푸시 금지
    - 반드시 `features/지원자이름` 브랜치 생성 후 해당 브랜치에서만 작업
    - 예시: `git checkout -b features/hong-gil-dong dev`

4. **개발 및 커밋**

    - 기능 단위로 커밋 (예: `feat: 책 추가 API 구현`)
    - 의미 없는 `fix`, `update` 단독 커밋 금지
    - 커밋 메시지는 목적이 드러나게 작성

5. **AI 활용**

    - **Cursor** 프롬프트 대화 내역을 export chat(MD 파일)로 저장
    - 해당 MD 파일을 브랜치에 커밋하여 제출

6. **REPORT.md 작성**

    - 개발 결과, 회고, 추가 메시지 등 자유롭게 작성
    - 실제 구현 결과, 느낀 점, 아쉬운 점, 본인만의 인사이트 등 자유롭게 기술

7. **제출**
    - 최종 제출은 개별 생성한 브랜치로 제출
    - dev 브랜치에 직접 push 시 금지

---

## 제출 파일 및 폴더 구조

-   `apps/frontend/` : Next.js 프론트엔드
-   `apps/backend/` : Express.js 백엔드
-   `PLANNING.md` : 기획서 (문제 선택, 핵심 기능, API 설계 등)
-   `REPORT.md` : 개발 결과/회고/메시지
-   (필수) **Cursor** 프롬프트 대화 내역(MD 파일)

# 📖 미션: 나만의 독서 도구 만들기

책을 좋아하는 사람들이 겪는 아래 문제 중 1~2개를 선택해, 본인만의 창의적이고 실용적인 도구를 만들어보세요.

-   읽고 싶은 책이 많은데 우선순위 결정을 못한다
-   읽은 책의 내용을 기억하지 못한다
-   친구들이 무엇을 읽는지 궁금하다
-   내 독서 패턴을 분석해보고 싶다

**예시**

-   책 우선순위 관리(관심도/긴급도/난이도 기반 추천)
-   독서 노트/검색/태그/필터
-   친구 독서 현황 피드/진행률
-   독서 통계/차트/인사이트
