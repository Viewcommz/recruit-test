# 📚 1시간 AI 개발 테스트 - 나만의 독서 도구

> **개발자 채용 1시간 테스트**  
> AI 도구를 활용한 풀스택 프로토타이핑 능력 평가

## ⏰ 테스트 개요

**소요시간**: 정확히 60분  
**목표**: AI 협업을 통한 창의적 문제해결 및 풀스택 구현 능력 평가

### 📊 평가 기준

-   ✅ **창의적 기획** (25%): 주어진 힌트로 독창적 기능 제안
-   ✅ **AI 활용도** (35%): Cursor, ChatGPT 등 AI 도구 활용 능력
-   ✅ **풀스택 구현** (30%): 프론트엔드 + 백엔드 연동 완성
-   ✅ **코드 품질** (10%): 가독성 및 구조

## 🎯 미션: "나만의 독서 도구"

### 주어진 조건

```
📖 상황: 당신은 책을 좋아하지만 항상 이런 불편함을 느낍니다:
   - 읽고 싶은 책은 많은데 뭘 먼저 읽을지 모르겠음
   - 읽은 책 내용이 기억나지 않음
   - 친구들이 뭘 읽는지 궁금함
   - 독서 패턴을 분석해보고 싶음

🤔 문제: 이 중에서 1-2개 문제를 해결하는 풀스택 웹 애플리케이션을 만들어주세요.
```

### 🚫 제약사항

-   **시간**: 60분 엄수
-   **범위**: 1-2개 핵심 기능만
-   **기술**: Next.js (프론트엔드) + Express.js (백엔드)
-   **데이터**: 메모리 저장 또는 간단한 JSON 파일
-   **디자인**: 최소한의 스타일링

### ✅ 필수 요구사항

1. **기획서 작성** (5분): 어떤 문제를 어떻게 해결할지 간단히 문서화
2. **풀스택 구현**: 프론트엔드 + 백엔드 API 연동
3. **작동하는 웹앱**: 실제 사용 가능한 수준
4. **AI 활용 증명**: 어떤 AI 도구를 어떻게 활용했는지 기록


## 💡 힌트 및 아이디어

### 풀스택 구현 예시

```
예시 1: "읽고 싶은 책 우선순위"
→ 프론트엔드: 책 입력 폼 + 우선순위 목록
→ 백엔드: POST /api/books, GET /api/books
→ 로직: 우선순위 계산 알고리즘

예시 2: "읽은 책 기억 도구"
→ 프론트엔드: 독서 노트 입력 + 검색 UI
→ 백엔드: POST /api/notes, GET /api/notes
→ 로직: 키워드 검색, 태그 필터링

예시 3: "친구들 독서 현황"
→ 프론트엔드: 친구 목록 + 독서 현황 피드
→ 백엔드: POST /api/friends, GET /api/status
→ 로직: 친구 관계, 독서 상태 관리

예시 4: "독서 패턴 분석"
→ 프론트엔드: 차트 표시 + 통계 대시보드
→ 백엔드: GET /api/stats, 데이터 집계
→ 로직: 독서 패턴 분석, 통계 계산
```

### 🤖 AI 활용 팁

```
Cursor/ChatGPT 프롬프트 예시:
"Express.js로 책 목록을 관리하는 API를 만들어줘.
POST /api/books로 책을 추가하고,
GET /api/books로 우선순위 순으로 정렬된 목록을 반환해줘.
메모리에 저장하면 돼."

"Next.js에서 위 API를 호출하는 컴포넌트를 만들어줘.
책 추가 폼과 목록 표시 기능이 필요해."

→ 이런 식으로 프론트엔드와 백엔드를 분리해서 요청
```
